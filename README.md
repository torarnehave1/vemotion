# Video Generator

A React + Vite application for generating MP4 videos on Cloudflare Workers, learning from Remotion's architecture but building a lightweight, serverless-friendly solution.

## Project Structure

```
video-generator/
├── src/                  # React frontend
│   ├── components/       # UI components
│   ├── lib/              # Utilities (auth, API)
│   ├── App.tsx           # Main app
│   └── main.tsx          # Entry point
├── worker/               # Cloudflare Worker (backend)
│   └── index.ts          # Worker API
├── public/               # Static assets
└── package.json          # Dependencies
```

## Features

- **React + Vite**: Fast development and build
- **Vegvisr UI Kit**: Consistent design system
- **Magic Link Auth**: Secure authentication via email
- **Cloudflare Workers**: Serverless API backend
- **Frame-based Rendering**: Learn from Remotion's approach
- **Composition as Data**: Store animations as JSON structures

## Development

### Install Dependencies
```bash
npm install
```

### Run Both Frontend & Worker
```bash
npm run dev:all
```

This runs:
- React dev server on `http://localhost:3002`
- Worker on `http://localhost:8790`

The Vite config proxies `/api/*` requests to the worker.

### Run Just Frontend
```bash
npm run dev
```

### Run Just Worker
```bash
npm run dev:worker
```

## Key Concepts

### Composition
A composition is a **data structure** (not React code) that describes what to render:

```typescript
{
  "duration": 5,
  "fps": 30,
  "width": 1280,
  "height": 720,
  "layers": [
    {
      "id": "text-1",
      "type": "text",
      "position": { "x": 100, "y": 100 },
      "properties": { "text": "Hello" },
      "animation": { /* keyframes */ }
    }
  ]
}
```

### Frame-Based Rendering
Videos are generated by:
1. Calculating each layer's state at each frame
2. Rendering the frame to canvas
3. Encoding all frames to MP4

### Worker API
The worker provides:
- `POST /api/video/create` - Queue a video render
- `GET /api/video/:id/status` - Check render progress
- `GET /api/video/:id/download` - Get MP4 download URL
- `GET /api/templates` - List video templates

## Learning from Remotion

### What Remotion Does
```
React Component → Headless Browser → Frames → FFmpeg → MP4
```

### Our Approach (Learning)
```
Composition Data → Browser Canvas → WASM Encoder → MP4
```

**Key differences:**
- No React bundling needed (simpler, faster)
- Data-driven instead of code-driven
- Browser does rendering (avoids headless browser)
- WASM FFmpeg for encoding (serverless compatible)

## Deployment

```bash
# Deploy worker to Cloudflare
npm run build:worker

# Build frontend
npm run build
```

Then deploy the `dist/` folder to Cloudflare Pages with the worker.

## Tech Stack

- **Frontend**: React 18, Vite, TypeScript, Tailwind CSS
- **UI**: vegvisr-ui-kit
- **Backend**: Hono, Cloudflare Workers
- **Auth**: Magic link via cookie.vegvisr.org
- **Storage**: R2 (videos), KV (cache), D1 (metadata)
